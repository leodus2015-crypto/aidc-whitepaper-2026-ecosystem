# 第1章 Agent 正在重塑每个企业

> 定位：生成式大语言模型横空出世后，Agent 概念也紧随其后，让 AI 从「对话」跃迁到「做事」阶段。意图理解、规划、记忆和工具调用，已经成为企业落地 AI 应用的标准模式。企业实现 Agentic AI 转型，应打造两类 Agent 和四项工程能力，并为 Agent 平台建设匹配的 IT 基础设施，相关阐述请参考本章节内容。

## 1.1 Agent 和大模型共生发展，携手将 AI 转化为生产力

### 关键信息
- [草稿] Agent 将大模型输出的 Token 转换成「生产力」，扩大了大模型应用场景的范围
-[草稿]Agent要做事、做正确的事，必须能够准确理解意图，记住历史，把复杂任务拆解成若干步骤合理规划，并调用合适的工具
- [草稿] AAgent 自身不断强化意图理解和规划能力，基础大模型也不断内化相关能力，两者相互依赖，共生发展

## 1.3 企业需构建具备四个工程能力的 Agent 平台

### 关键信息
- [草稿] 企业应打造岗位 Agent 和员工 Agent 两类 Agent
- [草稿] Agent 平台需具备四项工程能力，包括构建上下文工程、驾驭工程、多 Agent 协同工程和共生工程四个能力：
（1）上下文工程（Context Engineering），核心是解决上下文长度易爆炸，模型无记忆的问题；
（2）驾驭工程（Harness Engineering）核心问题是保证单 Agent 长时任务完成率；
（3）多Agent协同工程（Coordination Engineering）重点是多Agent大规模协同，实现蜂群作业；
（4）共生工程（Symbiosis Engineering）关注和周边环境的交互，让 Agent 从虚拟世界走向物理世界

状态：草稿
